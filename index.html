ساخت یک مخزن جدید

یک پوشه جدید بسازید، آنرا باز کنید و این دستور را اجرا کنید
git init
اینگونه یک مخزن گیت ایجاد می‌کنید.

[checkout] رونوشت گیری از مخزن

برای ساخت یک کپی کاری از مخزن، دستور زیر را اجرا کنید
git clone /path/to/repository
و اگر از مخزن ریموت و غیرمحلی استفاده می‌کنید دستور زیر را به کار گیرید
git clone username@host:/path/to/repository

جریان کار در گیت

مخزن محلی شما شامل ۳ درخت است که توسط گیت مدیریت می‌شود.اولی پوشه کاری Working Directory شما است که فایل‌های اصلی را نگه‌داری می‌کند.دومین پوشه Index است که به عنوان یک محیط واسط عمل می‌کند و در نهایت HEAD قرار دارد که شامل آخرین کامیت‌های شما است.

افزودن و کامیت کردن

شما می‌توانید تغییرات را با دستور زیر در Index قرار دهید.

git add <filename>

git add *

این اولین قدم از جریان کاری گیت است ولی برای کامیت کردن واقعی تغییرات باید از دستور زیر بهره برد.

git commit -m "Commit message"

حال کدهای شما در HEAD قرار دارند ولی هنوز در مخزن ریموت قرار نگرفته است.

ارسال تغییرات

اکنون تغییرات شما در HEAD پوشه کاری محلی شما قرار دارد برای ارسال آن‌ها به مخزن ریموت دستور زیر را اجرا کنید:
git push origin master
برای ارسال به یک شاخه دیگر به جای master نام آن شاخه را قرار دهید.
اگر شما از یک مخزن موجود کپی نگرفته‌اید،برای وصل کردن مخزن خود به یک مخزن ریموت باید آنرا با دستور زیر اضافه کنید.

git remote add origin <server>

حالا شما می‌توانید تغییرات را نیز به مخزن ریموت ارسال کنید.

branching [ساخت شاخه]

شاخه‌ها برای جداکردن توسعه بخش‌های گوناگون پروژه از یکدیگر استفاده می‌شوند.وقتی شما یک مخزن جدید ایجاد می‌کنید شاخه پیش‌فرض شما master است.

ساخت یک شاخه جدید به اسم "feature_x" و سوئیچ کردن به آن با دستور زیر
git checkout -b feature_x
برگشتن به شاخه master
git checkout master
حذف شاخه ساخته شده
git branch -d feature_x
شاخه ساخته شده تا به مخزن ریموت ارسال نشود توسط دیگران قابل دسترسی نیست.

git push origin <branch>

بروزرسانی و ادغام کردن

برای بروزکردن مخزن محلی خود با جدیدترین کامیت‌ها دستور زیر را

git pull

در پوشه کاری خود برای دریافت و ادغام تغییرات مخزن ریموت اجرا کنید.
برای ادغام یک شاخه با شاخه فعال به عنوان مثال master از دستور زیر

git merge <branch>

در هر دو حالت گیت سعی می‌کند به صورت اتوماتیک تغییرات را ادغام کنید ولی متاسفانه اینکار همیشه شدنی نیست و شما با مغایرت‌های برخورد می‌کنید. شما باید این مغایرت‌ها را که گیت نمایش می‌دهد به صورت دستی برطرف کرده و بعد از تغییرات آن‌‌ها را این گونه نشانه‌گذاری کنید

git add <filename>

قبل از عمل ادغام شما با دستور زیر می‌توانید تغییرات را مشاهده کنید.

git diff <source_branch> <target_branch>

tagging [تگ گذاری]

توصیه می‌شود که با انتشار نرم‌افزار آن‌ را تگ‌گذاری کنیم این موضوع مفهوم شناخته‌ شده‌ای در SVN است فرضآ اگر بخواهیم تگ 1.0.0 بسازیم باید دستور زیر را اجرا کنیم
git tag 1.0.0 1b2e1d63ff
که 1b2e1d63ff ده کاراکتر اول ‌آی‌دی کامیت شما است برای بدست آوردن آی‌دی کامیت از دستور زیر بهره ببرید
git log
شما همچنین می‌توانید از کاراکترهای کمتری از آی‌دی بهره ببرید ، تنها باید این آی‌دی یکتا باشد.

حذف تغییرات محلی

اگر شما کار اشتباهی انجام داده‌اید و قصد دارید آن اشتباه حذف شود دستور زیر را اجرا کنید

git checkout -- <filename>

این دستور تغییرات را با آخرین تغییراتی که در HEAD قرار دارد جایگزین می‌کند.

اگر قصد پاک کردن تمام تغییرات محلی را دارید باید آخرین تاریخچه موجود در مخزن ریموت را دریافت کنید و در شاخه master قرار دهید.
git fetch origin
git reset --hard origin/master

تذکرهای سودمند

کلاینت گرافیکی درونی گیت
gitk
خروجی‌های گیت را رنگی کنید
git config color.ui true
نمایش لاگ‌ها در یک خط برای هر کامیت
git config format.pretty oneline
از افزودن تعاملی بهره ببرد
git add -i
